Lists

A list is an abstract data type with a collection of items where each item holds a relative position with respect to the others. Lists can be argued to form the basis for other abstract data types like linked lists, stacks, arrays and queues.
Lists differ from stacks and queues in that additions and removals can be made at any position in the list. In lists, a value may occur more than once. Each instance of the value is considered a distinct item.

Note: Lists are a built in data structure for the Python programming language as it doesn't support arrays. Most other languages like C, C++ and Java provide arrays. Arrays are fixed size structures while lists are dynamic size structures.

There are two types of lists i.e unordered lists and ordered lists

Unordered Lists

In the implementation of the unordered list data type, the following operations may be used;

  • A constructor list() for creating an empty list.
  • is_empty() for testing whether or not a list is empty. It checks to see if the head of a list is a reference to none.
  • append(item) for appending an entity to the list
  • add(item) for adding an entity to the list. The easiest place to add an entity is at the head of the list.
  • pop(pos) for removing and returning the entity at position pos.
  • size() returns the number of entities in the list.
  • search(item) for searching for an entity in the list.
  • remove(item) for removing an entity from the list.

size(), search(item) and remove(item) all require traversing the list.
The unordered list is usually implemented as a singly or doubly linked list or as a dynamic array. Refer here for linked lists.
It can also be implemented as a binary search tree.

Ordered Lists

The Ordered List data type is called so because entities are ordered in a way that is reflective of their characteristic, for example ascending order, alphabetical order etc.

The operations on an ordered list are the same as those on the unordered list data type.

Some performance characteristics...

  • The time needed to get or set an individual item is constant, no matter what the size of the list is (time complexity is O(1)).
  • The time needed to insert an item depends on the size of the list, or more exactly, how many items that are to the right of the inserted item (O(n)). In other words, inserting items at the end is fast, but inserting items at the beginning can be relatively slow, if the list is large.
  • The time needed to sort a list varies; the worst case is O(n log n), but typical cases are often a lot better than that.
