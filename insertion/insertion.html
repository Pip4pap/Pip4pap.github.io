Insertion sort

The insertion sort, which is of O(n2) time complexity, works in a slightly different way compared to other sorting algorithms. It works in a way that it divides an unsorted array(list) in two parts with one part unsorted and the other part sorted which is maintained like that therefore an element to be inserted in the sorted part has to find in the appropriate place and is placed there. It consumes one input element in every iteration growing the sorted part by one element .This procedure is repeated until the whole array is sorted.

Consider the following example below that sorts the array in ascending order.

Algorithm

        Step 1 − If it is the first element, it is already sorted. return 1;
        Step 2 − Pick next element
        Step 3 − Compare with all elements in the sorted sub-list
        Step 4 − Shift all the elements in the sorted sub-list that is greater than the 
                 value to be sorted
        Step 5 − Insert the value in the appropriate position
        Step 6 − Repeat until list is sorted

The array is searched sequentially and unsorted items are moved and inserted into the sorted part. This algorithm is not suitable for large data sets as its average and worst case complexity are of Ο(n2), where n is the number of items(input size).
