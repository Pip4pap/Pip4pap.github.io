Stacks

A stack can based be described as an ordered collection of items where addition and removal of items takes place at the same end. It uses the principle of LIFO(Last in First out).The top of the stack is the end where items are removed from and added to the stack. The base is the other end of the stack.
The items close to the base have been in the stack the longest while the recently added items are the ones in close to the top of the stack. A brief example: Assume a clean table. Place plates one at a time on top of each other. When removing them one at a time, the order of removal is exactly the reverse order of replacement.
Conclusion, in a stack the order of insertion is the reverse of the order of removal.
The stack operations include;

  • Push: When this method is invoked it adds an item on top of the stack
  • Pop: The pop method removes the top element from the stack whenever it is called
  • Peek: This stack method returns top element of the stack
  • View: Returns a list of all elements of the stack when called
  • Size: Returns a number of elements of the stack
