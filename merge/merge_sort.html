Merge Sort

The merge sort is part of a group of algorithms that employ a paradigm known as divide and conquer. This paradigm is based on recursion. The worst-case time complexity of merge sort is O(nlog n) but it's space complexity is O(n). The problem is broken down into subproblems similar to the original problem. The subproblems are solved recursively and the solutions to the subproblems combined to solve the original problem.

A base case is a subarray containing fewer than two elements. A base case is already sorted since a subarray with no elements or one element is already sorted.

Consider an array holding elements [p..r] where p and r are indexes
We divide and conquer only when p < r.
Here's how merge sort uses divide and conquer

  • Divide by finding the number q of the position midway between p and r. This is done by dividing the sum of p and r by 2 and rounding down.
  • Conquer by recursively sorting the subarrays in each of the two subproblems created by the divide step. That is, recursively sort the subarray array[p..q] and recursively sort the subarray array[q+1..r].
  • Merge by combining the two sorted subarrays back into the single sorted subarray array[p..r]..

Algorithm

    Step 1 − if it is only one element in the list it is already sorted, return.
    Step 2 − divide the list recursively into two halves until it can no more be divided.
    Step 3 − merge the smaller lists into new list in sorted order.
